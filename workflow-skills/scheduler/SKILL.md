# Scheduler (智能调度)

**后台自动运行 → Claude + MiniMax 开会分析 → 留痕 → 调整方向**

## 核心功能

### 1. 后台自动触发

```
任务执行时自动触发:
├── 任务开始时
├── 任务进行中 (每5分钟)
├── 任务完成时
└── 检测到关键节点
```

### 2. Claude + MiniMax 会议流程

```
任务执行中...
     │
     │ 后台静默运行
     ▼
┌────────────────────────────────┐
│  🤖 Claude Code 分析          │
│     - 当前进度                 │
│     - 优化空间                 │
│     - 风险点                  │
└──────────────┬───────────────┘
               │
               ▼ 生成分析报告
┌────────────────────────────────┐
│  🌟 MiniMax 开会讨论          │
│     - 评估当前方向             │
│     - 提出优化建议             │
│     - 制定新计划              │
└──────────────┬─────────────────┘
               │
               ▼ 会议记录
┌────────────────────────────────┐
│  📝 留痕到日志文件             │
│     - 时间戳                   │
│     - 会议内容                │
│     - 决策结果                │
└──────────────┬───────────────┘
               │
               ▼
┌────────────────────────────────┐
│  🔄 调整 auto-pilot          │
│     - 方向更新                │
│     - 任务重排               │
│     - 策略优化                │
└──────────────┬───────────────┘
               │
               ▼
     继续执行 (无感知)
```

### 3. 会议留痕

```
📋 会议记录 (自动保存)
├── 时间: 2024-01-15 14:30:00
├── 参与者: Claude Code + MiniMax
├── 分析内容:
│   ├── 当前任务进度: 60%
│   ├── 检测到问题: 某个API响应慢
│   └── 优化建议: 切换并行请求
├── 决策:
│   ├── 调整执行策略: 并行化
│   └── 优先级重排: 任务C提前
└── 执行时间: 立即生效
```

### 4. 动态调整

```
原计划: A → B → C
     │
     │ MiniMax 会议分析
     ▼
优化后: A → C → D (跳过B，更高效)
     │
     │ 自动调整 + 留痕
     ▼
继续执行新方向 + 记录日志
```

---

## 留痕机制

### 日志文件

| 文件 | 位置 | 内容 |
|------|------|------|
| 会议记录 | `./scheduler/meetings/` | 每次开会内容 |
| 分析报告 | `./scheduler/analysis/` | Claude 分析结果 |
| 决策日志 | `./scheduler/decisions/` | 调整记录 |

### 记录格式

```json
{
  "timestamp": "2024-01-15T14:30:00Z",
  "type": "scheduling_meeting",
  "participants": ["Claude Code", "MiniMax"],
  "analysis": {
    "progress": "60%",
    "issues": ["API响应慢"],
    "suggestions": ["切换并行请求"]
  },
  "decisions": {
    "strategy_change": "并行化",
    "task_reorder": ["A", "C", "D"]
  }
}
```

---

## 无感知运行

| 特点 | 说明 |
|------|------|
| 后台运行 | 用户无感知 |
| 不打断 | 不影响当前任务 |
| MiniMax 参与 | Gemini 一起开会分析 |
| 自动留痕 | 会议记录保存到文件 |
| 实时调整 | 自动优化方向 |

---

## 命令

| 命令 | 功能 |
|------|------|
| `scheduler on` | 开启后台调度 |
| `scheduler off` | 关闭 |
| `分析` | 手动触发分析 |
| `优化方向` | 重新规划 |
| `开会` | 手动触发会议 + 留痕 |
| `查看会议记录` | 查看历史会议 |

---

## 自动触发

当 auto-pilot 执行任务时：
- 每5分钟后台 Claude + MiniMax 开会分析
- 关键节点自动规划 + 留痕
- 完成后自动复盘 + 记录
