# Project Planner (项目规划器)

**遵循 s03 原理：没有计划就走到哪算哪**

> *"没有计划的 agent 走哪算哪"*

## 核心原理

### s03 核心：TodoWrite + 步骤规划

```
用户: "帮我做一个股票网站"

规划流程:
1. 理解需求 (1分钟)
2. 列出步骤 (TodoWrite)
3. 按步骤执行
4. 每步确认
```

**完成率对比**:
- ❌ 无规划: ~30% 完成
- ✅ 有规划: ~80% 完成

---

## 规划工作流

### Step 1: 理解需求 (30秒)

问关键问题：
- 目标是什么？
- 受众是谁？
- 有什么约束？
- 预期成果？

### Step 2: 列出步骤 (TodoWrite)

```markdown
## 📋 任务清单

- [ ] 步骤1: 需求分析
- [ ] 步骤2: 技术选型
- [ ] 步骤3: 架构设计
- [ ] 步骤4: 代码实现
- [ ] 步骤5: 测试部署
```

### Step 3: 按序执行

- 每步完成后打勾 ✅
- 遇到问题记录 ❌
- 随时可调整后续步骤

### Step 4: 阶段检查

每完成 2-3 个小任务，做一次检查：
- 方向对吗？
- 需要调整吗？

## 6 阶段模板

### Stage 1: 需求理解
- 明确目标
- 确定约束
- 定义产出

### Stage 2: 技术选型
- 语言/框架
- 工具/API
- Skills 匹配

### Stage 3: 架构设计
- 模块划分
- 数据流
- 接口设计

### Stage 4: 实现规划
- 任务拆分
- 里程碑
- 依赖关系

### Stage 5: 执行迭代
- 按计划编码
- 测试调试
- 优化完善

### Stage 6: 总结复盘
- 文档化
- 经验总结

| 命令 | 功能 |
|------|------|
| "规划" / "计划" | 启动规划流程 |
| "步骤" / "todo" | 查看任务清单 |
| "下一步" | 执行下一任务 |
| "完成" | 标记任务完成 |
