# Project Planner (项目规划器)

**文档主导工作流 - 信息充分原则**

> *"给AI的信息越充分，产出越好"*
> *"文档成为开发者与项目之间的源代码"*

---

## 文档主导工作流 (默认)

```
📄 文档主导 (所有任务默认):

  1. 需求发散
     - 收集背景信息
     - 明确目标
     - 列出约束

  2. 写PRD/Spec
     - 功能描述
     - 技术方案
     - 验收标准

  3. Plan + Review
     - 列出实现步骤 (TodoWrite)
     - 用户确认方向

  4. 实现
     - 按步骤执行
     - 关注架构 ✓

  5. 测试迭代
     - AI自动生成测试
     - 修复问题

  6. 人工Review
     - 用户确认
     - 交付
```

---

## 何时用原型主导 (仅限小功能)

```
🎨 原型主导 (仅限):
  - 需求模糊不清
  - 小功能/实验性
  - 快速验证概念

用法: 明确告知 "用原型"
```

---

## Plan First - 信息收集

```
收到任务 → 先问清楚:

  1. 项目背景是什么?
  2. 目标用户是谁?
  3. 有什么约束/限制?
  4. 预期交付什么?
  5. 验收标准?

信息越充分 → 产出越好
```

---

## TodoWrite 步骤规划

```markdown
## 📋 任务清单

### 需求分析
- [ ] 收集背景信息
- [ ] 明确功能需求
- [ ] 确定约束条件

### 文档
- [ ] 编写PRD
- [ ] 技术方案设计

### 实现
- [ ] 步骤1: xxx
- [ ] 步骤2: xxx

### 测试
- [ ] AI生成测试
- [ ] 修复问题

### Review
- [ ] 用户确认
- [ ] 交付
```

---

## 关注架构 (非细节)

```
实现时关注:
  ✓ 模块划分
  ✓ 接口设计
  ✓ 系统交互
  ✓ 数据流

不纠结:
  ✗ 变量命名
  ✗ 代码格式
  ✗ 语法细节
```

## 命令

| 命令 | 功能 |
|------|------|
| "规划" | 启动文档主导模式 |
| "原型" | 切换原型主导(小功能) |
| "下一步" | 执行下一任务 |
| "查看进度" | 显示任务列表 |

---

## 设计原则

1. **文档默认**: 所有任务先用文档
2. **信息充分**: 收集足够上下文
3. **TodoWrite**: 步骤可视化
4. **架构优先**: 关注系统交互
