# Auto-Pilot (自动驾驶)

**遵循 s03 规划 + s08 后台 + 文章优化**

> *"没有计划的agent走哪算哪"* + *"慢操作丢后台"* + *"给AI的信息越充分越好"*

---

## 核心理念（来自文章）

### 1. 计划先行 (Plan First)
- 每次任务 → 先分析
- **信息越充分 → 产出越好**

### 2. 后台不阻塞 (s08)
- 长时间任务 → 丢后台
- agent 继续 → 下一步

### 3. 关注架构
- 系统交互点 ✓
- 代码主干 ✓

---

## 自动检测并行任务

### 并行检测规则

| 关键词 | 自动触发 |
|--------|----------|
| "和" / "与" | 分割任务 → parallel-worker |
| "并行" | 直接触发 parallel-worker |
| "同时" | 分割任务 → parallel-worker |
| "多个" | 分割任务 → parallel-worker |
| "后台" | 触发 async-runner |

### 自动并行流程

```
用户: "查前列腺癌和肺癌论文"

1. auto-pilot 检测到 "和"
2. → 收集信息 (Plan First)
3. → 自动分割任务:
   - 任务1: 前列腺癌论文
   - 任务2: 肺癌论文
4. → 触发 parallel-worker
5. → 并行执行
6. → 合并结果
```

| Level | 名称 | 容忍度 |
|-------|------|--------|
| 0 | 手动 | 0% - 全部确认 |

| 2 | 常规 | 50% - 常规自动 |
| 3 | 信任 | 75% - 大多数自动 |
| 4 | 完全 | 100% - 无干扰 |
