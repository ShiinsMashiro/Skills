# Auto-Pilot (自动驾驶)

**遵循 s03 规划 + s08 后台原理**

> *"没有计划的 agent 走哪算哪"* + *"慢操作丢后台, agent 继续想下一步"*

## 核心原理

### s03: 先规划后执行

```
用户: "自动开发项目"

1. 规划 (TodoWrite): 列出步骤
2. 执行: 按步骤进行
3. 检查: 每步确认方向
```

### s08: 后台不阻塞

```
长时间任务 → 丢后台
agent 继续 → 下一步
完成后 → 通知注入
```

---

## 自动检测并行任务

**auto-pilot 会自动检测任务类型**：

### 并行检测规则

| 关键词 | 自动触发 |
|--------|----------|
| "和" / "与" | 分割任务 → parallel-worker |
| "并行" | 直接触发 parallel-worker |
| "同时" | 分割任务 → parallel-worker |
| "多个" | 分割任务 → parallel-worker |
| "后台" | 触发 async-runner |

### 自动并行流程

```
用户: "查前列腺癌和肺癌论文"

1. auto-pilot 检测到 "和"
2. → 自动分割任务:
   - 任务1: 前列腺癌论文
   - 任务2: 肺癌论文
3. → 触发 parallel-worker
4. → 并行执行两个搜索
5. → 合并结果
```

### 后台任务检测

```
用户: "查论文，顺便做其他事"

1. auto-pilot 检测到 "顺便"
2. → 触发 async-runner
3. → 任务1 丢后台
4. → 继续处理任务2
```

## 权限级别

| Level | 名称 | 容忍度 |
|-------|------|--------|
| 0 | 手动 | 0% - 全部确认 |
| 1 | 先问 | 25% - 危险操作确认 |
| 2 | 常规 | 50% - 常规自动 |
| 3 | 信任 | 75% - 大多数自动 |
| 4 | 完全 | 100% - 无干扰 |

## 工作流循环

```
┌─────────────────────────────────────────┐
│  AUTO-PILOT LOOP                       │
├─────────────────────────────────────────┤
│                                         │
│  1. 接收任务                            │
│     ↓                                  │
│  2. TodoWrite 规划                     │
│     ↓                                  │
│  3. 执行步骤                            │
│     ↓                                  │
│  4. 后台任务? → 丢后台继续             │
│     ↓                                  │
│  5. 检查结果 → 更新状态                 │
│     ↓                                  │
│  6. 完成? → 结束                       │
│     ↓                                  │
│  7. 继续下一任务 ←←←←                  │
└─────────────────────────────────────────┘
```

### 执行示例

```
用户: "查前列腺癌和肺癌论文"
       ↓
自动检测: "和" → 并行任务
       ↓
parallel-worker 分割:
  - 子任务1: 前列腺癌
  - 子任务2: 肺癌
       ↓
并行执行 → 合并结果
```
